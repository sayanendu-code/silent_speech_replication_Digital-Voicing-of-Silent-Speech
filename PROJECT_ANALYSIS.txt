  ═══════════════════════════════════════════════════════════════════
  PAPER 1: EMNLP 2020 — "Digital Voicing of Silent Speech"
  ═══════════════════════════════════════════════════════════════════

  INPUT FEATURES (hand-designed, 112-dim):
  ─────────────────────────────────────────
  Raw EMG (8ch × 1000Hz)
       │
       ▼
  Butterworth HP filter (2 Hz) + Notch (60 Hz + harmonics)
       │
       ▼
  Triangular split → x_low (< 134 Hz) + x_high (> 134 Hz)
       │
       ▼
  Frame: 27ms window, 10ms stride → 100 Hz feature rate
       │
       ├── Time-domain features per channel (5):
       │     mean(x_low²), mean(x_low), mean(x_high²), mean(|x_high|), ZCR(x_high)
       │
       └── STFT magnitude features per channel (9):
             16-point FFT → 9 magnitude bins
       │
       ▼
  Total: (5 + 9) × 8 channels = 112 features per frame

  TRANSDUCER (trainable):
  ───────────────────────
  EMG features (B, T, 112)
       │
       ▼
  [+ Session Embedding]  ← 32-dim per session, appended at every timestep
       │                    (B, T, 144)
       ▼
  Dropout(0.5)           ← TRAINABLE: nothing, just regularization
       │
       ▼
  BiLSTM Layer 1         ← TRAINABLE: 4 × 1024 × (1024+144) + 4 × 1024 × 1024
       │                    = ~8.8M params (forward) + same (backward)
       ▼
  Dropout(0.5)
       │
       ▼
  BiLSTM Layer 2         ← TRAINABLE: 4 × 1024 × (2048+1024) × 2
       │                    ~25M params
       ▼
  Dropout(0.5)
       │
       ▼
  BiLSTM Layer 3         ← TRAINABLE: same as layer 2
       │                    ~25M params
       ▼
  Dropout(0.5)
       │
       ▼
  Linear(2048 → 26)      ← TRAINABLE: 2048 × 26 + 26 = ~53K params
       │
       ▼
  Predicted MFCCs (B, T, 26) @ 100 Hz

  TOTAL TRANSDUCER PARAMS: ~60-70M (estimate)

  VOCODER (WaveNet, separately trained):
  ───────────────────────────────────────
  Gold/Predicted MFCCs (T, 26) @ 100 Hz
       │
       ▼
  BiLSTM(512, bidirectional)   ← TRAINABLE: conditioning network
       │
       ▼
  Linear(1024 → 128)           ← TRAINABLE
       │
       ▼
  ConvTranspose1d(upsample)    ← 100 Hz → 16,000 Hz (factor 160)
       │
       ▼
  WaveNet Core                 ← TRAINABLE:
       │  16 dilated causal conv layers
       │  max dilation: 128
       │  residual channels: 64
       │  skip channels: 256
       │  gated activation + conditioning
       ▼
  Softmax(256)                 ← μ-law quantized output
       │
       ▼
  Audio waveform @ 16 kHz


  ═══════════════════════════════════════════════════════════════════
  PAPER 2: ACL 2021 — "An Improved Model for Voicing Silent Speech"
  ═══════════════════════════════════════════════════════════════════

  WHAT CHANGED (3 key modifications):
  ────────────────────────────────────

  CHANGE 1 — Input: Hand-designed features → Learned Conv features
  ┌──────────────────────────────────────────────────────────────┐
  │  BEFORE (Paper 1):                                          │
  │    EMG → filter → 112-dim hand-designed features @ 100 Hz   │
  │                                                              │
  │  AFTER (Paper 2):                                           │
  │    Raw EMG (8ch × 1000 Hz)                                  │
  │         │                                                    │
  │         ▼  only: HP 2Hz + notch 60Hz + resample 800Hz       │
  │         │   + magnitude /10                                  │
  │         ▼                                                    │
  │    ResConvBlock 1 (768ch, stride 2) → 400 Hz                │
  │         ▼                                                    │
  │    ResConvBlock 2 (768ch, stride 2) → 200 Hz                │
  │         ▼                                                    │
  │    ResConvBlock 3 (768ch, stride 2) → 100 Hz                │
  │         ▼                                                    │
  │    [+ Session Embed 32→768, added per timestep]              │
  │         ▼                                                    │
  │    Learned features (B, T, 768) @ 100 Hz                    │
  │                                                              │
  │  Each ResConvBlock:                                         │
  │    input → Conv1d(k=3) → BatchNorm → ReLU                  │
  │         → Conv1d(k=3) → BatchNorm → [+ shortcut] → ReLU    │
  │    shortcut: Conv1d(k=1, stride=2) for downsampling         │
  └──────────────────────────────────────────────────────────────┘

  CHANGE 2 — Backbone: BiLSTM(1024×3) → Transformer(768, 6 layers)
  ┌──────────────────────────────────────────────────────────────┐
  │  BEFORE (Paper 1):                                          │
  │    3× BiLSTM, 1024 hidden, dropout 0.5                      │
  │                                                              │
  │  AFTER (Paper 2):                                           │
  │    6× Transformer encoder layers                             │
  │      - model dim: 768                                       │
  │      - heads: 8                                              │
  │      - FFN dim: 3072                                        │
  │      - dropout: 0.1                                          │
  │      - RELATIVE position embeddings (not absolute)           │
  │      - k=100 (1 second window each direction)               │
  │      - attention beyond k is zeroed out                      │
  └──────────────────────────────────────────────────────────────┘

  CHANGE 3 — Loss: Pure MSE → MSE + Auxiliary Phoneme CTC loss
  ┌──────────────────────────────────────────────────────────────┐
  │  BEFORE (Paper 1):                                          │
  │    Loss = MSE(predicted_mfcc, target_mfcc)                  │
  │                                                              │
  │  AFTER (Paper 2):                                           │
  │    Additional linear head on top of Transformer:             │
  │      → Linear(768 → n_phonemes) → Softmax                  │
  │                                                              │
  │    Combined cost for alignment:                              │
  │      δ'[i,j] = ||A_V[i] - Â_S[j]||² − λ·P_V[i]ᵀ·log(P̂_S[j]) │
  │      λ = 0.1                                                │
  │                                                              │
  │    Phoneme labels from Montreal Forced Aligner               │
  │    Phoneme head is DISCARDED after training                  │
  └──────────────────────────────────────────────────────────────┘

  OTHER CHANGES (Paper 2):
    - Optimizer: Adam → AdamW (weight decay 1e-7)
    - LR warmup: 500 batches linear warmup to 1e-3
    - Epochs: up to 200 → fixed 80
    - Batching: concatenate-and-reshape (2s chunks, 256s per batch)
    - Alignment: simplified to predicted-audio only (no separate CCA step)
    - Vocoder: same WaveNet (unchanged)


  ═══════════════════════════════════════════════════════════════════
  DISSERTATION 2022 — "Voicing Silent Speech" (Final Model)
  ═══════════════════════════════════════════════════════════════════

  WHAT CHANGED FROM PAPER 2:
  ──────────────────────────

  CHANGE — Vocoder: WaveNet → HiFi-GAN
  ┌──────────────────────────────────────────────────────────────┐
  │  BEFORE (Papers 1 & 2):                                     │
  │    WaveNet: autoregressive, sample-by-sample                │
  │    VERY slow inference (minutes per utterance)               │
  │                                                              │
  │  AFTER (Dissertation):                                      │
  │    HiFi-GAN: feed-forward GAN-based vocoder                │
  │    Fine-tuned from UNIVERSAL_V1 pretrained model            │
  │    Fine-tuned for 75,000 steps                               │
  │    Input: predicted spectrograms (not gold)                  │
  │    FAST inference (real-time or faster)                      │
  │                                                              │
  │  Result: better audio quality + massively faster synthesis   │
  └──────────────────────────────────────────────────────────────┘

  ADDITION — Silent Speech Recognition (direct EMG → text)
  ┌──────────────────────────────────────────────────────────────┐
  │  Same Conv + Transformer transducer architecture             │
  │  BUT: output head replaced with CTC decoder                 │
  │    → predicts character sequence directly                    │
  │    → no vocoder needed                                      │
  │    → KenLM language model for decoding                      │
  │  WER: ~28% (open vocabulary, text output)                   │
  └──────────────────────────────────────────────────────────────┘

  ═══════════════════════════════════════════════════════════════════
  PAPER 1 TRAINING (what we replicated):
  ═══════════════════════════════════════════════════════════════════

  PHASE 1 — Initial Alignment (before training starts)
  ─────────────────────────────────────────────────────

  For each parallel utterance (E_S, E_V, A_V):

    Silent EMG ──────────► extract 112-dim features ──► E'_S (M, 112)
    Vocalized EMG ───────► extract 112-dim features ──► E'_V (N, 112)
    Vocalized Audio ─────► extract 26 MFCCs ──────────► A'_V (N, 26)

    E'_S and E'_V ───────► DTW (Euclidean distance)
         │                      │
         ▼                      ▼
    alignment[i] → j        (which frame in E'_V matches frame i in E'_S)
         │
         ▼
    Warped targets:  Ã'_S[i] = A'_V[ alignment[i] ]
                     Now we have (E'_S, Ã'_S) pairs!

  PHASE 2 — CCA Refinement (still before training)
  ─────────────────────────────────────────────────

    Collect ALL aligned pairs from Phase 1:
      (E'_S[aligned], E'_V[warped]) across entire dataset
         │
         ▼
    Fit CCA (15 components):
      Learn projections P_S and P_V
      that maximize correlation between silent & vocalized EMG
         │
         ▼
    Re-run DTW with CCA-projected distance:
      δ_CCA[i,j] = ||P_S · E'_S[i] − P_V · E'_V[j]||
         │
         ▼
    Updated warped targets: Ã'_S (better aligned)

  PHASE 3 — Training Loop (per epoch)
  ────────────────────────────────────

    ┌────────────────────────────────────────────────────────────┐
    │  TRAINING BATCH (mixed from 3 sources):                   │
    │                                                            │
    │  Source A: (E'_S, Ã'_S, session_silent)                   │
    │    ← silent EMG + warped audio targets                     │
    │                                                            │
    │  Source B: (E'_V, A'_V, session_vocalized)                │
    │    ← vocalized parallel EMG + real audio                   │
    │                                                            │
    │  Source C: (E'_V, A'_V, session_nonparallel)              │
    │    ← non-parallel vocalized EMG + real audio               │
    │                                                            │
    │  All mixed in each batch, differentiated by session IDs    │
    └────────────────────────────────────────────────────────────┘
              │
              ▼
    Transducer:  EMG + SessionEmbed → BiLSTM×3 → Linear → Â'
              │
              ▼
    Loss = MSE(Â', target)     ← target is Ã'_S or A'_V
              │
              ▼
    Adam optimizer, lr=0.001, halved every 5 epochs no improvement
              │
              ▼
    Validation loss on silent EMG test → checkpoint best model

  PHASE 4 — Predicted Audio Re-alignment (epoch 5+, every 5 epochs)
  ──────────────────────────────────────────────────────────────────

    Partially-trained transducer
         │
         ▼
    Run inference on all silent EMG → get predicted MFCCs Â'_S
         │
         ▼
    New DTW cost:
      δ_full[i,j] = δ_CCA[i,j] + 10 · ||Â'_S[i] − A'_V[j]||
                                   ↑ λ=10
         │
         ▼
    Re-compute ALL alignments → new warped targets Ã'_S
         │
         ▼
    Continue training with updated targets

  WAVENET TRAINING (separate):
  ────────────────────────────

    Gold vocalized audio (16kHz) + Gold MFCCs (26-dim, 100Hz)
         │
         ▼
    Teacher forcing:
      Input:  gold audio shifted by 1 sample
      Target: μ-law quantized next sample (256 classes)
      Conditioning: gold MFCCs (upsampled to 16kHz)
         │
         ▼
    Cross-entropy loss, batch_size=1 (memory constraints)
         │
         ▼
    NOT trained on predicted MFCCs → avoids error compounding

  ═══════════════════════════════════════════════════════════════════

  Key difference: NO separate CCA step. Alignment uses predicted audio only.

    For silent utterances:
      Run transducer → Â'_S

# ┌─────────────────────────────────────────────────────────────────────┐
# │  7. WHAT MADE EACH IMPROVEMENT WORK — Ablation Summary             │
# └─────────────────────────────────────────────────────────────────────┘

  PAPER 1 → PAPER 2 (68.0% → 42.2% = 25.8% absolute improvement):
  ─────────────────────────────────────────────────────────────────

  Improvement                  Isolated WER   Contribution
  ──────────────────────────  ────────────   ─────────────
  Learned conv features         45.2%         −3.0% vs full (explains 12%)
  Transformer (vs LSTM)         46.0%         −3.8% vs full (explains 15%)
  Phoneme auxiliary loss        51.7%         −9.5% vs full (explains 37%)
  All three combined            42.2%         full model

  The phoneme loss was the single biggest contributor.
  It provides the model with supervision about WHAT sounds are being
  produced, not just what they sound like acoustically.

  PAPER 2 → DISSERTATION (42.2% → ~36% = ~6% improvement):
  ──────────────────────────────────────────────────────────

  Primary change: WaveNet → HiFi-GAN
    - Better audio quality (GAN training produces more natural speech)
    - Faster inference (real-time vs minutes per utterance)
    - Fine-tuned on PREDICTED spectrograms (adapts to model errors)

  Alignment                Predicted-audio DTW only       DTW + CCA + audio
  Optimizer                AdamW + warmup                 Adam
  Batching                 Concat-reshape (2s chunks)     Standard padding

  Evaluation ASR           DeepSpeech 0.7                 Whisper / DeepSpeech
  Text recognition         Yes (CTC, ~28% WER)            No
  Pre-trained available    Yes (Zenodo 6747411)           Train from scratch

  TO REACH CURRENT BEST (~36% WER), you would need to:
    1. Replace BiLSTM with Transformer + relative position embeddings
    2. Replace hand-designed features with learned Conv blocks
    3. Add phoneme auxiliary loss (requires Montreal Forced Aligner)
    4. Replace WaveNet with HiFi-GAN
    5. Use AdamW with warmup schedule
    6. Use concat-reshape batching strategy

# ┌─────────────────────────────────────────────────────────────────────┐
# │  11. SUMMARY — THE BIG PICTURE                                     │
# └─────────────────────────────────────────────────────────────────────┘

  2020: "Can we voice silent speech at all?"
        → Yes! 3.6% WER on closed vocab, 68% on open vocab.
        → Key insight: transfer audio targets from vocalized to silent.

  2021: "Can we make it actually intelligible in open vocabulary?"
        → 42.2% WER — huge jump from learned features + Transformer + phonemes.
        → Key insight: let the model learn its own features from raw EMG.

  2022: "Can we make it practical and also do recognition?"
        → 36% WER voicing, 28% WER recognition.
        → Key insight: HiFi-GAN makes real-time synthesis possible.

  The fundamental challenge remains: silent EMG signals are qualitatively
  different from vocalized EMG (no vocal fold vibration, different
  articulation patterns to suppress sound). This domain gap is why
  the alignment/transfer machinery exists at all.
